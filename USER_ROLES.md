# 👥 User Roles & Workflows

CityCare is built around three distinct user personas, each with a specialized interface.

### 🦸 The Citizen
The heart of the platform. Citizens are the "eyes and ears" of the city.
- **AI-Powered Reporting**: A conversational AI agent that extracts issue details and locations automatically.
- **Manual Reporting**: Full control over report details with an interactive map picker.
- **Duplicate Detection**: Prevents spam by identifying similar nearby reports before submission.
- **Impact Points**: Gamified system where citizens earn points for verified resolutions.
- **Status Tracking**: Real-time progress bars and resolution timelines for every report.
- **Re-issue Protocol**: Ability to reopen "Resolved" issues if the work was unsatisfactory.

### 🛡️ The System Admin
The strategic layer. Admins manage the city's resources and prioritize response.
- **Tactical Command Center**: A high-level overview of all active reports with priority filtering.
- **Resource Deployment**: Assigning specific field workers to tasks based on department.
- **Impact Analytics**: Real-time charts (Recharts) showing resolution velocity and category distribution.
- **Escalation Protocol**: Automated system to upgrade the priority of long-pending issues.
- **Security Audit**: Moderation tools to handle flagged reports and maintain community trust.

### 👷 The Field Worker
The execution layer. Workers receive "Missions" and provide technical evidence.
- **Mission Dashboard**: A focused list of assigned tasks with location data.
- **Safety Protocol**: Mandatory safety checklists (PPE, area cordoning) before submitting work.
- **Evidence Upload**: Capturing "After" photos to prove resolution.
- **Shift Management**: Integrated clock-in/out system to track operational hours.
